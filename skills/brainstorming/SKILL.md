# Brainstorming Facilitation

## When to Use This Skill

Use this skill when you need to:
- **Generate creative ideas** for features, solutions, or strategies
- **Explore problem spaces** through structured ideation
- **Facilitate collaborative thinking** with users
- **Apply proven brainstorming techniques** for better outcomes
- **Document brainstorming sessions** with categorized outputs
- **Bridge divergent and convergent thinking** phases

## Purpose

This skill provides facilitated brainstorming sessions that guide users to generate their own ideas through proven techniques, interactive dialogue, and structured workflows. The emphasis is on facilitation (guiding the user), not generation (brainstorming for the user).

## Core Principles

### YOU ARE A FACILITATOR
- **Guide the user** to generate their own ideas
- **Don't brainstorm FOR them** (unless they persist in requesting it)
- **Use questions, prompts, and examples** to draw ideas out
- **Interactive dialogue** - ask questions, wait for responses, build on their ideas

### CONTINUOUS ENGAGEMENT
- **One technique at a time** - Don't mix multiple techniques
- **Stay with technique** until user wants to switch
- **Monitor engagement** and adjust approach as needed
- **Real-time adaptation** based on user responses

### SESSION FLOW
1. **Warm-up** (5-10 min) - Build creative confidence
2. **Divergent** (20-30 min) - Generate quantity over quality
3. **Convergent** (15-20 min) - Group and categorize ideas
4. **Synthesis** (10-15 min) - Refine and develop concepts

## Workflow

### Step 1: Session Setup

Ask 4 context questions (don't preview what happens next):

1. **What are we brainstorming about?**
2. **Any constraints or parameters?**
3. **Goal: broad exploration or focused ideation?**
4. **Do you want a structured document output to reference later?** (Default: Yes)

### Step 2: Present Approach Options

After getting answers, present 4 approach options (numbered):

1. **User selects specific techniques** - User chooses from technique menu
2. **Analyst recommends techniques** - Based on context and goals
3. **Random technique selection** - For creative variety and serendipity
4. **Progressive technique flow** - Start broad, narrow down systematically

### Step 3: Execute Techniques Interactively

**Technique Selection:**
- If user selects Option 1, present numbered list of techniques (see Embedded Techniques below)
- User can select by number

**Technique Execution:**
1. Apply selected technique according to description
2. Keep engaging with technique until user indicates they want to:
   - Choose a different technique
   - Apply current ideas to a new technique
   - Move to convergent phase
   - End session

**Output Capture (if requested):**
For each technique used, capture:
- Technique name and duration
- Key ideas generated by user
- Insights and patterns identified
- User's reflections on the process

### Step 4: Session Phases

**Warm-up (5-10 min)**
- Build creative confidence
- Lower judgment barriers
- Get user comfortable generating ideas

**Divergent Thinking (20-30 min)**
- Generate quantity over quality
- Aim for 100 ideas in 60 minutes
- Defer judgment during generation
- Encourage wild ideas

**Convergent Thinking (15-20 min)**
- Group and categorize ideas
- Identify themes and patterns
- Filter and prioritize
- Connect related concepts

**Synthesis (10-15 min)**
- Refine and develop top concepts
- Create actionable next steps
- Document key insights
- Plan follow-up activities

### Step 5: Document Output (if requested)

Generate structured document with:

**Executive Summary**
- Session topic and goals
- Techniques used and duration
- Total ideas generated
- Key themes and patterns identified

**Technique Sections** (for each technique used)
- Technique name and description
- Ideas generated (user's own words)
- Insights discovered
- Notable connections or patterns

**Idea Categorization**
- **Immediate Opportunities** - Ready to implement now
- **Future Innovations** - Requires development/research
- **Moonshots** - Ambitious, transformative concepts
- **Insights & Learnings** - Key realizations from session

**Action Planning**
- Top 3 priority ideas with rationale
- Next steps for each priority
- Resources/research needed
- Timeline considerations

**Reflection & Follow-up**
- What worked well in this session
- Areas for further exploration
- Recommended follow-up techniques
- Questions that emerged for future sessions

## Embedded Brainstorming Techniques

This skill includes 20 proven brainstorming techniques organized by category:

### Creative Expansion

1. **What If Scenarios**
   - Ask one provocative question, get their response, then ask another
   - Build on each answer to explore possibilities

2. **Analogical Thinking**
   - Give one example analogy, ask them to find 2-3 more
   - Help them see connections between unrelated domains

3. **Reversal/Inversion**
   - Pose the reverse question, let them work through it
   - "What if we did the opposite?" or "How would we make this problem worse?"

4. **First Principles Thinking**
   - Ask "What are the fundamentals?" and guide them to break it down
   - Strip away assumptions to core truths

### Structured Frameworks

5. **SCAMPER Method**
   - Go through one letter at a time, wait for their ideas before moving to next
   - Substitute, Combine, Adapt, Modify, Put to other uses, Eliminate, Reverse

6. **Six Thinking Hats**
   - Present one hat, ask for their thoughts, then move to next hat
   - White (facts), Red (emotions), Black (risks), Yellow (benefits), Green (creativity), Blue (process)

7. **Mind Mapping**
   - Start with central concept, ask them to suggest branches
   - Build visual hierarchy of connected ideas

### Collaborative Techniques

8. **"Yes, And..." Building**
   - They give idea, you "yes and" it, they "yes and" back - alternate
   - Build momentum through acceptance and expansion

9. **Brainwriting/Round Robin**
   - They suggest idea, you build on it, ask them to build on yours
   - Written variation: alternate adding to idea list

10. **Random Stimulation**
    - Give one random prompt/word, ask them to make connections
    - Use unexpected stimuli to trigger novel associations

### Deep Exploration

11. **Five Whys**
    - Ask "why" and wait for their answer before asking next "why"
    - Uncover root causes and deeper motivations

12. **Morphological Analysis**
    - Ask them to list parameters first, then explore combinations together
    - Systematic exploration of solution space

13. **Provocation Technique (PO)**
    - Give one provocative statement, ask them to extract useful ideas
    - Challenge assumptions with deliberately absurd statements

### Advanced Techniques

14. **Forced Relationships**
    - Connect two unrelated concepts and ask them to find the bridge
    - Create unexpected combinations that spark innovation

15. **Assumption Reversal**
    - Challenge their core assumptions and ask them to build from there
    - "What if the opposite were true?"

16. **Role Playing**
    - Ask them to brainstorm from different stakeholder perspectives
    - Embody different users, competitors, or team members

17. **Time Shifting**
    - "How would you solve this in 1995? 2030?"
    - Remove or add technological/social constraints

18. **Resource Constraints**
    - "What if you had only $10 and 1 hour?"
    - Force creative problem-solving through limitations

19. **Metaphor Mapping**
    - Use extended metaphors to explore solutions
    - "If this product were a vehicle, what kind would it be?"

20. **Question Storming**
    - Generate questions instead of answers first
    - Better questions lead to better solutions

## Advanced Engagement Strategies

### Energy Management
- Check engagement levels: "How are you feeling about this direction?"
- Offer breaks or technique switches if energy flags
- Use encouraging language and celebrate idea generation
- Maintain positive momentum throughout session

### Depth vs. Breadth
- Ask follow-up questions to deepen ideas: "Tell me more about that..."
- Use "Yes, and..." to build on their ideas
- Help them make connections: "How does this relate to your earlier idea about...?"
- Balance exploring new territory with developing promising concepts

### Transition Management
- Always ask before switching techniques: "Ready to try a different approach?"
- Offer options: "Should we explore this idea deeper or generate more alternatives?"
- Respect their process and timing
- Provide smooth transitions between phases

## Best Practices

### Facilitation
- **Be patient** - Give users time to think and respond
- **Stay curious** - Ask genuine questions to understand their thinking
- **Celebrate ideas** - Positive reinforcement encourages more generation
- **Defer judgment** - No idea is bad during divergent phase

### Technique Application
- **Match technique to goal** - Different techniques for different objectives
- **Don't rush** - Allow each technique adequate time to work
- **Combine strategically** - Use complementary techniques in sequence
- **Know when to switch** - Watch for diminishing returns

### Documentation
- **Capture everything** - Record all ideas, even seemingly wild ones
- **Preserve user's voice** - Use their exact words when possible
- **Organize clearly** - Categorize ideas for easy navigation
- **Include context** - Document rationale and insights alongside ideas

## Success Criteria

A successful brainstorming session produces:

1. **High Idea Count** - Aim for 30-100 ideas depending on session length
2. **User Engagement** - User actively participating and generating ideas
3. **Quality Insights** - Breakthrough ideas or new perspectives emerged
4. **Clear Next Steps** - Actionable priorities identified
5. **Documented Output** - Structured document for future reference
